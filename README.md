# Swarm Brain

Командная база знаний с AI-поиском и интеграцией митингов. Работает в Telegram и Claude Desktop, живёт на Supabase Edge Functions.

---

## Цель проекта

Собрать в одном месте всё, что знает команда — заметки, документы, договорённости, итоги встреч, ссылки — и сделать эту информацию мгновенно доступной через естественный язык. Бот работает там, где команда уже общается (Telegram), не требует переключения контекста.

---

## Функциональность

### База знаний
- **Добавление** — текст, ссылки, документы (TXT, MD, CSV, JSON, XLSX, PDF), фото, голосовые сообщения
- **Хранение файлов** — PDF и документы сохраняются в Supabase Storage, ссылка на оригинал доступна через поиск
- **Семантический поиск** — поиск по смыслу через векторные эмбеддинги (OpenAI `text-embedding-3-small`)
- **AI-ответы** — бот отвечает на вопросы по содержимому базы (GPT-4o)
- **Автообработка медиа** — голос транскрибируется (Whisper), фото описывается (GPT-4o Vision), URL — содержимое страницы извлекается и сохраняется

### Встречи (Read.ai)
- Автоматический вебхук: как только встреча завершается в Read.ai — транскрипт сохраняется в базу с подтверждением
- Подтверждение перед сохранением — бот присылает превью с кнопками: сохранить / изменить название / изменить дату
- Просмотр встреч через `/meetings` с кнопками: детали, выгрузить файл, удалить
- Теги/страны и участники привязываются к встрече через inline-кнопки

### Задачи
- Просмотр задач с фильтрами по исполнителю, тегу/стране, сроку
- Смена статуса через inline-кнопки (open → in_progress → done / cancelled)

### Пользователи и доступ
- Белый список: только разрешённые Telegram-аккаунты имеют доступ к боту
- Профили пользователей: имя, роль, рынки, телефон, email

### MCP-сервер (Claude Desktop)
База знаний как MCP-инструменты для работы в Claude Desktop:

| Инструмент | Что делает |
|---|---|
| `search_knowledge` | Семантический поиск по базе |
| `get_entry` | Полный текст записи по ID |
| `add_knowledge` | Добавить запись с тезисами |
| `get_tasks` | Задачи с фильтрами |
| `get_meetings` | Последние встречи |
| `get_users` | Список команды с профилями |
| `list_entries` | Ревизия: список записей с фильтрами по источнику, типу, дате |
| `delete_entry` | Удалить запись (и файл из Storage) |
| `update_entry` | Обновить содержимое или метаданные записи |

## Технический стек

| Слой | Технология |
|------|-----------|
| Runtime | Deno (Supabase Edge Functions) |
| База данных | Supabase (PostgreSQL + pgvector) |
| Хранилище файлов | Supabase Storage (bucket: swarm_drive) |
| AI | OpenAI GPT-4o, Whisper, text-embedding-3-small |
| Интерфейс | Telegram Bot API |
| Митинги | Read.ai (OAuth2 + webhook) |

## Структура проекта

```
supabase/functions/
├── swarm-bot/              # Telegram-бот
│   ├── index.ts            # Роутинг сообщений и cron-триггеры
│   ├── lib/                # Утилиты
│   │   ├── supabase.ts     # Supabase клиент
│   │   ├── telegram.ts     # Telegram API
│   │   ├── openai.ts       # OpenAI (embeddings, chat, whisper)
│   │   ├── storage.ts      # База знаний + Supabase Storage
│   │   ├── readai.ts       # Read.ai API
│   │   └── types.ts        # TypeScript типы
│   └── handlers/           # Обработчики сценариев
│       ├── knowledge.ts    # Добавление и поиск по базе
│       ├── media.ts        # Файлы, голос, фото, URL
│       ├── meetings.ts     # Встречи и транскрипты
│       ├── tasks.ts        # Задачи
│       ├── users.ts        # Пользователи и профили
│       ├── digest.ts       # Персональный дайджест
│       └── help.ts         # Текст справки
├── swarm-mcp/              # MCP-сервер для Claude Desktop
├── read-ai-auth/           # OAuth2 авторизация Read.ai
└── read-ai-webhook/        # Вебхук для приёма встреч из Read.ai
```
